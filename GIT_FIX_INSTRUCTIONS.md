# Инструкция по исправлению проблемы с большими файлами

## Что было сделано:

1. ✅ Большие видео файлы удалены из истории Git через `git filter-branch`
2. ✅ Добавлены в `.gitignore` для предотвращения повторного добавления
3. ✅ Очищены резервные копии и оптимизирован репозиторий

## Следующие шаги:

**ВАЖНО:** История Git была переписана, поэтому нужен force push.

### Вариант 1: Force push (если вы единственный разработчик)

```bash
git push --force origin master
```

### Вариант 2: Force push with lease (безопаснее, если есть другие разработчики)

```bash
git push --force-with-lease origin master
```

## Предупреждение:

- Force push перезапишет историю на удаленном сервере
- Если другие разработчики работают с репозиторием, им нужно будет:
  1. Удалить локальную копию
  2. Клонировать заново: `git clone <repository-url>`

## Проверка:

После push проверьте, что файлы больше не в репозитории:
```bash
git ls-tree -r HEAD --name-only | grep "assets/videos/.*\.mp4"
```

Должны остаться только файлы из папки `optimized/`.
